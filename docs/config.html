配置说明

概述

svs-mock 通过 mock_certs.toml 配置服务端口、日志级别、信任根证书、签名密钥和加密密钥。服务启动时自动加载此配置，构建内存中的证书仓库。

查找顺序

优先级方式说明
1环境变量 SVS_MOCK_CONFIG指定配置文件绝对路径
2当前工作目录 ./mock_certs.toml启动进程时 shell 的 pwd

[server] 服务配置

[server]
port      = 9000     # HTTP 监听端口（默认 9000）
log_level = "info"   # 日志级别：debug / info / warn / error

[[trusted_roots]] 信任根证书

配置信任的根证书，用于 ValidateCert 验证证书链和 ExportCert 导出证书。

[[trusted_roots]]
name = "测试根 CA"              # 名称（仅用于日志标识）
cert = "MIIBxTCCAWu..."        # DER 格式证书的 base64 编码
可配置多个 [[trusted_roots]]。证书加载后同时建立 subject 和 serialNumber 两个索引，供 ExportCert 的 certSubjectBase64 和 certSN 两种查询方式使用。

[[signing_keys]] 签名密钥

配置 SM2 签名密钥对，供 SignData 和 SignMessage 接口使用。

[[signing_keys]]
index       = 1                       # 密钥索引（对应请求中的 keyIndex）
pin         = "12345678"              # 密钥 PIN（对应请求中的 password）
private_key = "cc54df...（64位hex）"  # SM2 私钥（32字节 hex）
cert        = "MIIBxTCCAWu..."       # 签名证书 DER base64
PIN 验证：SignData 和 SignMessage 请求中的 password 必须与配置中的 pin 完全匹配，否则返回 ERR_KEY_AUTH (0x04000006)。

[[enc_keys]] 加密密钥

配置 SM2 加密密钥对，供 envelopeDec 接口使用。

[[enc_keys]]
index       = 1                       # 密钥索引（对应请求中的 keyIndex）
private_key = "aa11bb...（64位hex）"  # SM2 加密私钥（32字节 hex）
cert        = "MIIBxTCCAWu..."       # 加密证书 DER base64

[server]
port      = 9000
log_level = "info"

# ─── 信任根证书（可配多个）───
[[trusted_roots]]
name = "测试根 CA"
cert = """
MIIBxTCCAWugAwIBAgIBATAKBggqgRzPVQGDdTBEMQswCQYDVQQGEwJDTjEQMA4G
A1UECAwHQmVpamluZzEQMA4GA1UEBwwHQmVpamluZzERMA8GA1UECgwITW9ja0NB
...
"""

[[signing_keys]]

MIIBxTCCAWugAwIBAgIBATAKBggqgRzPVQGDdTBEMQswCQYDVQQGEwJDTjEQMA4G

生成测试证书

项目提供证书生成工具，可生成符合要求的 SM2 自签名证书和密钥：

# 从源码运行证书生成工具
cd 0029-svs-mock
cargo run --example gen_test_data

# 输出 SM2 密钥对（hex）和自签名证书（DER base64）
# 将输出内容填入 mock_certs.toml 对应字段即可
⚠️ 安全警告：配置文件中的私钥和证书仅用于开发测试。严禁在生产环境使用示例密钥。如需部署到公网，请使用自行生成的测试密钥，或配合组织的 CA 签发正式证书。
