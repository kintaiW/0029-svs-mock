接口文档

接口概览

所有接口均为 POST 方法，Content-Type: application/json。成功时 respValue = 0（GM_SUCCESS），失败返回对应错误码。

URL 路径和字段名与纽创信安 XC-M30W 签名验签服务器一致。

请求/响应格式

通用响应结构：

{
  "respValue": 0,      // 0=成功，非零=错误码
  ...                  // 接口特有的返回字段
}

数据编码约定：

  • 二进制数据（明文、密文、签名值、摘要）：Base64 编码
  • 证书：DER 格式的 Base64 编码
  • 证书序列号（certSN）：十六进制 字符串

证书管理

POST /ExportCert

按主题或序列号导出证书。

请求

{
  "certSubjectBase64": "...",  // 证书主题 base64（二选一）
  "certSN": "01"               // 证书序列号 hex（二选一）
}

响应

{
  "respValue": 0,
  "certBase64": "MIIB..."      // DER 证书 base64
}
POST /ValidateCert

验证证书有效性：有效期 + 信任锚签名。

请求

{
  "certBase64": "MIIB..."      // 待验证证书 DER base64
}

响应

{
  "respValue": 0               // 0=有效，非零见错误码
}
验证逻辑：检查 notBefore ≤ 当前时间 ≤ notAfter，并验证是否被信任根证书签发。不检查 CRL/OCSP。
POST /ParseCert

解析证书指定字段。

请求

{
  "certBase64": "MIIB...",
  "infoType": 1                // 字段类型（见下表）
}
infoType返回内容
1版本号 (version)
2序列号 (serialNumber)
5颁发者 (issuer DN)
6有效期起始 (notBefore)
7有效期截止 (notAfter)
8使用者 (subject DN)
0x31签名算法 OID
0x35主题公钥 (hex)
0x36主题公钥算法 OID

摘要运算

POST /Digest

SM3 摘要计算。支持传入公钥 + userId 进行 Z 值预计算（用于 SM2 签名前的预哈希）。

请求

{
  "inData": "aGVsbG8=",        // 待计算数据 base64
  "publicKey": "...",          // 可选：SM2 公钥 base64（用于 Z 值）
  "userId": "..."              // 可选：用户标识 base64
}

响应

{
  "respValue": 0,
  "hashData": "..."            // SM3 摘要 base64（32字节）
}

数字签名

POST /SignData

使用服务端私钥进行 SM2 P1 签名（裸签名，DER 编码 r||s）。

请求

{
  "keyIndex": 1,               // 私钥索引（对应 mock_certs.toml 中的 signing_keys）
  "password": "12345678",      // 密钥 PIN
  "inData": "aGVsbG8="        // 待签名数据 base64
}

响应

{
  "respValue": 0,
  "signData": "...",           // DER 编码签名值 base64
  "certBase64": "MIIB..."     // 签名证书 base64
}
签名流程：先以默认 userId（1234567812345678）计算 Z 值，再 e = SM3(Z || inData)，最后 SM2 签名。
POST /VerifySignedData

使用证书公钥验证 SM2 P1 签名。

请求

{
  "certBase64": "MIIB...",     // 签名证书
  "inData": "aGVsbG8=",       // 原始数据
  "signData": "..."            // 签名值 base64
}

响应

{
  "respValue": 0               // 0=验签成功
}
POST /SignMessage

生成 PKCS7/CMS SignedData 格式的签名消息（附带签名证书链）。

请求

{
  "keyIndex": 1,
  "password": "12345678",
  "inData": "aGVsbG8="
}

响应

{
  "respValue": 0,
  "signedMessage": "..."       // CMS SignedData DER base64
}
POST /VerifySignedMessage

验证 PKCS7/CMS SignedData 签名消息。

请求

{
  "inData": "aGVsbG8=",
  "signedMessage": "..."       // CMS SignedData DER base64
}

响应

{
  "respValue": 0               // 0=验签成功
}

数字信封

POST /envelopeEnc

SM2+SM4 数字信封加密：随机生成 SM4 会话密钥，SM4-CBC 加密数据，SM2 加密会话密钥。

请求

{
  "certBase64": "MIIB...",     // 接收方加密证书
  "inData": "aGVsbG8="        // 待加密数据 base64
}

响应

{
  "respValue": 0,
  "envelopedData": "..."       // 数字信封 base64
}
POST /envelopeDec

SM2+SM4 数字信封解密：SM2 解密会话密钥，SM4-CBC 解密数据。

请求

{
  "keyIndex": 1,               // 加密私钥索引
  "envelopedData": "..."       // 数字信封 base64
}

响应

{
  "respValue": 0,
  "outData": "aGVsbG8="       // 解密后数据 base64
}
