快速开始

下载预编译程序

前往 GitHub Releases 下载对应平台的发布包：

svs-mock-v0.1.0-linux-x86_64.tar.gz
  ├── svs-mock              # 可执行文件（单文件，无运行时依赖）
  ├── mock_certs.toml       # 示例配置（含测试证书和密钥）
  ├── examples/
  │   └── gen_test_data.rs  # 证书/密钥生成工具源码
  └── README.md

部署运行

# 1. 解压
tar -xzf svs-mock-v0.1.0-linux-x86_64.tar.gz
cd svs-mock-v0.1.0-linux-x86_64

# 2. 确保 mock_certs.toml 在当前目录（或用环境变量指定）
ls mock_certs.toml

# 3. 直接运行（默认监听 0.0.0.0:9000）
./svs-mock

# 或：指定配置文件路径
SVS_MOCK_CONFIG=/etc/svs/mock_certs.toml ./svs-mock

# 后台运行
nohup ./svs-mock > svs-mock.log 2>&1 &
公网部署：服务默认监听 0.0.0.0:9000，可直接通过 IP:9000 从外网访问。如需修改端口，编辑 mock_certs.toml 中的 [server] port 字段。建议配合 Nginx 反向代理或防火墙策略使用。

验证服务

服务启动后，用 curl 快速验证：

# SM3 摘要（最简单的测试）
curl -s -X POST http://localhost:9000/Digest \
  -H "Content-Type: application/json" \
  -d '{"inData":"aGVsbG8="}' | python3 -m json.tool

# 预期返回：
# {
#   "respValue": 0,
#   "hashData": "..."   (base64 编码的 SM3 摘要)
# }

# SM2 签名（需要配置 signing_keys）
curl -s -X POST http://localhost:9000/SignData \
  -H "Content-Type: application/json" \
  -d '{"keyIndex":1,"password":"12345678","inData":"aGVsbG8="}' | python3 -m json.tool

# 证书导出
curl -s -X POST http://localhost:9000/ExportCert \
  -H "Content-Type: application/json" \
  -d '{"certSN":"01"}' | python3 -m json.tool

源码编译

# 环境要求：Rust 1.75+
git clone https://github.com/kintaiW/0029-svs-mock.git
cd 0029-svs-mock

# 编译（release 模式，优化体积和性能）
cargo build --release
# 输出：target/release/svs-mock

# 运行
cargo run --release

生成测试证书

项目包含证书生成工具，可生成适用于 mock_certs.toml 的测试证书和密钥：

# 生成测试数据（SM2 密钥对 + 自签名证书）
cargo run --example gen_test_data

# 输出的密钥和证书可直接填入 mock_certs.toml

环境变量

变量说明默认值
SVS_MOCK_CONFIG指定配置文件路径./mock_certs.toml
RUST_LOG日志级别info（在 mock_certs.toml 中配置）

systemd 服务（可选）

在 Linux 服务器上可配置为 systemd 服务实现开机自启：

# /etc/systemd/system/svs-mock.service
[Unit]
Description=SVS Mock Server (GM/T 0029-2014)
After=network.target

[Service]
Type=simple
User=svs
WorkingDirectory=/opt/svs-mock
ExecStart=/opt/svs-mock/svs-mock
Environment=SVS_MOCK_CONFIG=/opt/svs-mock/mock_certs.toml
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable svs-mock
sudo systemctl start svs-mock
sudo systemctl status svs-mock
